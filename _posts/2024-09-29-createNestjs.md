# 1. Nestjs란?

<br>

express와 같이 nodejs 기반의 프레임워크입니다. 기본적으로 제공하는 기능이 다양하고, 형식이 정해져 있으며, 의존성 주입을 통해 유지보수가 쉽도록 구성되어 있습니다.

<br>

# 2. nestjs 설치하기

> 1. 프로젝트를 저장할 폴더를 생성합니다.<br>※ 이미 있다면 생략합니다.
> 2. vscode 화면 좌측 상단 - 파일 폴더 열기 - 해당 폴더 선택<br>※ 이미 선택되어 있다면 생략합니다.
> 3. `` ctrl + `  `` 버튼을 눌러 터미널을 열어줍니다.
> 4. 그리고 터미널에 `npm i @nestjs/core @nestjs/common @nestjs/platform-express reflect-metadata typescript`를 입력하여 nestjs 프로젝트 폴더를 생성하고, 그 외 필요한 라이브러리까지 설치합니다.
>    > - @nestjs/core : 데코레이터로 사용하는 함수들의 클래스 등 공통 기능들이 들어 있습니다.
>    > - @nestjs/commen : 가드, 미들웨어, 파이프 등을 만드는 핵심 기능들이 들어 있습니다.
>    > - @nestjs/platform-express : http 요청/응답을 감싸는 익스프레스의 req, res 객체를 netsjs에서 사용할 수 있게 해주는 라이브러리입니다.
>    > - reflect-metadata : 데코레이터 사용 시 필요한 라이브러리입니다.
>    > - typescript : 자바스크립트에 기반한 nodejs 역시 변수의 타입에 엄격하지 않은 편입니다. 타입이 자유롭다는 것은 장점도 있지만, 개발자가 본래 의도하지 않은 타입의 변수가 저장될 수 있다는 단점도 있습니다. 그러한 nodejs 코드에 타입을 지정할 수 있도록 하는 라이브러리입니다.

# 3. nestjs 초기 세팅하기

## 3-1. tsconfig.json

<br>

<p>프로젝트 폴더의 루트 디렉터리(폴더에 들어가자마자 바로 보이는 위치)에 "tsconfig.json" 파일을 생성하고 다음과 같이 입력합니다.</p>

```json
{
  "compilerOptions": {
    "module": "CommonJS",
    "target": "ESNext",
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```

<br>

## 3-2. nestjs식 파일 작명

<br>

<p>이름을 정하는 것은 아주 중요한 일입니다.</p>

<p>자신은 물론, 협업하는 타인도 쉽게 인식할 수 있도록 해야 하고, 프로그램에 따라 파일의 이름을 엄격하게 관리하여 폴더나 파일의 이름이 정해진 기준에 조금이라도 맞지 않으면 오류를 발생시키기도 합니다.</p>

<p>nestjs에서는 구현하고자 하는 기능별로 파일을 모아두는 폴더를 "모듈"이라고 부르고, 그 모듈 안에 구현하고자 하는 기능을 "컴포넌트"라고 합니다.</p>

<p>그리고 파일의 이름은 "모듈명.컴포넌트명.ts"라고 짓는 것이 관례입니다.</p>

<p>예를 들어, 모듈명이 "A"이고, 컴포넌트명이 "B"라면 파일의 이름은 "A.B.ts"가 되겠습니다.</p>

<br>

## 3-3. 폴더 구성

<br>

<p>프로젝트 폴더의 루트 디렉터리(폴더에 들어가자마자 바로 보이는 위치)에 "src" 폴더를 만들고, 그 안에 "hello" 폴더를 만듭니다. 주요 기능에 관한 것들은 모두 "src" 폴더에 들어갑니다.</p>

<p>사실 간단하게 할 거라서 "hello" 폴더는 안 만들어도 상관 없지만, 이해를 돕기 위해 이렇게 구성합니다.</p>

<br>

# 4. nestjs 앱 만들기

## 4-1. main.ts 파일 만들기

<br>

<p>src 폴더 안에 main.ts" 파일을 만들고, 다음의 내용을 입력합니다.</p>

```java
import { NestFactory } from "@nestjs/core"; // (1)
import { HelloModule } from "hello/hello.module"; // (2)

async function bootstrap() {
  const app = await NestFactory.create(HelloModule); // (3)
  await app.listen(3000, () => {
    console.log("서버시작");
  });
}

bootstrap();
```

<br>

<p>그렇게 생소하지 않은 구성입니다. nodejs 때나 express 때에 비해 크게 달라지지 않았습니다.</p>

| 번호 | 설명                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| (1)  | `require("http")` 대신 `"@nestjs/core"` 에서 가져온 `NestFactory` 를 사용합니다. `"@nestjs/core"`는 아까 전에 설치한 라이브러리입니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| (2)  | `"hello/hello.module"` 에서 `HelloModule`를 가져옵니다. "/" 앞에 hello는 우리가 방금 만든 hello 폴더를 의미하고, 뒤에 있는 "hello.module"은 우리가 곧 만들 파일입니다.<br><br> 자바의 스프링이나 파이썬의 장고 같은 프레임워크들과 달리, nestjs는 자신이 만든 파일을 다른 곳에서 import 할 수 있게 만들어 주려면 `module.ts`라는 파일을 만들어서 거기에 등록해야 합니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| (3)  | `async - await` 은 한 세트로, 비동기적 함수를 동기적으로 구현하기 위한 것입니다. 이것은 매우 길고 복잡한 내용이기 때문에 간단하게만 설명하겠습니다.<br><br>기본적으로 코드는 위에서부터 순차적으로 실행되지만, 그렇다고 해서 아래의 코드가 '위의 코드가 종료되기'까지 기다려주는 것은 아닙니다(비동기적). 그저 실행되는 순서만 차례대로일 뿐입니다.<br><br>하지만 이러면 문제가 발생합니다. 아래의 코드가 위의 코드의 결과를 사용해야 한다면 어떻게 될까요? 위의 코드의 결과가 나오기도 전에, 아래의 코드가 실행되어 버린다면, 아래의 코드는 필요한 값을 전달 받지 못하였으니 당연히 오류가 발생할 것입니다.<br><br>이러한 문제를 피하기 위해 결과가 나오기까지 기다리도록(동기적) 할 필요가 있습니다. 이러한 상황에서 비동기적으로 실행되는 함수를 동기적으로 바꾸기 위해 사용하는 것이 바로 `async - await` 입니다.<br><br>대부분이 코드에서는 이런 문제가 발생하지 않지만, 가끔 문제를 일으키는 코드가 있으니, 그게 어떤 것인지 궁금하시다면 나중에 따로 조사하시는 편이 좋습니다. |

## 4-2. hello.controller.ts 파일 만들기

<br>

<p>hello 폴더 안에 "hello.controller.ts" 파일을 만들고, 다음의 내용을 입력합니다.</p>

```java
import { Controller, Get } from "@nestjs/common"; // (1)

@Controller() // (2)
export class HelloController {
  @Get() // (3)
  hello() {
    return "안녕하세요! NestJS로 만든 첫 애플리케이션입니다.";
  }
}
```

<br>

| 번호 | 설명                                                                                                                                                                                                                                                        |
| ---- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| (1)  | `"@nestjs/common"` 에서 `Controller` 데코레이션과 `Get` 데코레이션을 가져와서 사용합니다. 자바 - 스프링부트에서는 어노테이션이라고 부르는 것입니다. 파일 이름이 controller라서 controller 역할을 하는 게 아니라 이것을 붙여야 그 역할을 수행할 수 있습니다. |
| (2)  | `@Controller` 데코레이션을 붙여서 Controller라는 역할을 지정해줍니다. Controller는 웹에서 온 사용자의 요청에 대응하는 역할을 말합니다.                                                                                                                      |
| (3)  | `@Get` 데코레이션을 붙여서 Get 요청에 대응하게 합니다.<br><br> `@Controller` 가 붙은 class 안에서만 사용할 수 있으며, 사용자로부터 Get 요청이 들어오면, 이 메서드가 실행되어 사용자에게 응답합니다.                                                         |

<br>

## 4-3. hello.module.ts 파일 만들기

<br>

<p>hello 폴더 안에 "hello.module.ts" 파일을 만들고, 다음의 내용을 입력합니다.</p>

```java
import { Module } from "@nestjs/common"; // (1)
import { HelloController } from "hello/hello.controller"; // (2)

// (3)
@Module({
  controllers: [HelloController],
})
export class HelloModule {} // (4)
```

<br>

| 번호 | 설명                                                                                                                                                                                                                                                                                                          |
| ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| (1)  | `"@nestjs/common"` 에서 `Module` 데코레이션을 가져와서 사용합니다. 파일 이름이 module이라서 module 역할을 하는 게 아니라 이것을 붙여야 그 역할을 수행할 수 있습니다.                                                                                                                                          |
| (2)  | `"hello/hello.controller"` 안에 있는 `HelloController`라는 클래스를 가져옵니다. 위에서 @Contoller를 붙인 그 클래스 이름입니다.<br><br>"/" 앞에 hello는 우리가 방금 만든 hello 폴더를 의미하고, 뒤에 있는 "hello.controller"은 우리가 만든 파일 이름입니다.                                                    |
| (3)  | `@Module` 데코레이션을 사용하면서 구체적으로 어떤 것들을 모듈 안에 등록할 지 입력합니다. 이렇게 등록해야만 HelloModule이라는 패키지에 담아서 외부로 내보낼 수 있습니다.<br><br>이 형식은 정해진 것이므로 임의로 수정해서는 안 됩니다.<br><br>여기서는 controllers에 HelloController 하나만 담아서 내보냅니다. |
| (4)  | 실질적인 기능은 `@Module` 데코레이션을 사용하면서 등록한 내용들이 전부이지만, 이것을 외부로 내보내기 위해서는 클래스라는 형태가 필요하기 때문에 클래스로 지정할 필요가 있습니다.<br><br>클래스로 지정하면서 export 해주면 이제 외부에서 사용할 수 있게 됩니다.                                                |

# 5. 웹 서버 실행하기

<br>

<p>타입스크립트로 만든 프로그램을 실행하기 위해서는 ts-node-dev를 붙여야 합니다.</p>

<p>일단 터미널에 실행시키겠다고 입력하면, 설치 여부를 자동으로 파악하고, 설치하지 않았다면 설치할 것인지 묻습니다.</p>

> 1. `` ctrl + `  `` 버튼을 눌러 터미널을 열어줍니다.
> 2. 터미널에 `npx ts-node-dev src/main.ts`를 입력합니다.
> 3. 설치할 것인지 물어보면 "y"를 입력합니다.
>    > - 이미 설치되어 있다면 물어보지 않습니다.
> 4. 인터넷 창을 열어서 주소창에 localhost:3000을 입력하여 정상적으로 실행되면 성공입니다.
