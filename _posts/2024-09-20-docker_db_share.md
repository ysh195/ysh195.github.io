# 도커로 DB 공유하기

이것은 프로젝트 과정에서 제가 겪었던 실수입니다.
<br>
혹시 저와 같은 생각을 하시는 분이 있다면 이 글을 보고 헛수고하지 않으셨으면 합니다.
<br>

**도커만으로는 DB와 그 데이터를 공유할 수 없습니다.**

하지만 다른 활용과 의도를 위해 도커 내 DB를 세팅하신다면 의미가 있을 수도 있습니다.

## 1. 발단

프로젝트를 위해 다수의 인원이 하나의 DB를 공유해야 하는 상황이 발생했습니다.
<br>
이 때, 제 생각은 다음과 같았습니다.

> 1. 해결 방법은 2가지<br>① 클라우드 DB를 사용한다. ② 도커로 DB를 공유한다.
> 2. 1번이 가장 깔끔하지만, 비용의 문제가 있고, 새로운 DB의 사용법을 익혀야 한다는 부담이 있다.
> 3. 그리고 그것이 프로젝트의 참여자 모두에게 적용된다면 프로젝트의 진행 측면에서도 손실일 수 있다.
> 4. 도커를 사용한다면, 기존의 DB를 도커를 통해 사용할 뿐이기 때문에 그러한 부담이 크게 줄어들고, 도커를 실행시킬 호스트가 항상 컴퓨터를 켜놔야 한다는 점만 빼면 비용도 발생하지 않는다.
> 5. 그러니까 도커를 쓰자.

## 2. 전개 과정

### 1단계 - 도커에 DB 컨테이너 세팅하기

<br>
도커 터미널에 아래와 같이 입력합니다.

```java
docker pull mysql --(1)
docker image ls
docker run --name mysql -d -p [사용할 포트]:3306 -e MYSQL_ROOT_PASSWORD=1111 mysql --(2)
docker ps
docker exec -it mysql /bin/bash --(3)
mysql -u root -p --(4)
password: 1111 --(5)
GRANT ALL PRIVILEGES ON *.* TO 'root'@'%'; --(6)
FLUSH PRIVILEGES; --(7)
CREATE DATABASE [DB이름]; --(8)
```

<br>

| 구분 | 설명                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ---- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| (1)  | 도커에 mysql을 이미지로 다운 받습니다. mysql 이미지 다운 기능은 도커에서 공식적으로 지원하는 것이며, mysql 이외의 다른 DB도 가능합니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| (2)  | mysql이라는 이름의 이름을 mysql이라는 이름의 컨테이너로 만들어서 실행합니다. 다른 DB를 다운 받았다면 그것에 맞춰야 합니다.<br><br>여기서 포트 번호가 가장 중요합니다.<br>":" 앞의 [사용할 포트]는 앞으로 컨테이너 내의 mysql에 접근할 때 사용할 포트 번호입니다. 팀원들과 남는 포트 번호가 무엇인지 상의가 필요합니다.<br>":" 뒤의 3306은 고정입니다. 도커에서 mysql에 접근하기 위해 사용되는 포트는 3306으로 자동으로 지정되기 때문에 만약 3306 이외의 다른 포트로 설정할 경우 연결이 되지 않습니다.<br><br>동시에, mysql의 초기세팅을 진행하여 root 계정의 비밀번호를 설정합니다. 비밀번호는 보다 복잡하게 설정하는 것을 추천합니다. |
| (3)  | 실행중인 mysql 컨테이너 내부의 터미널에 접속합니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| (4)  | mysql에 root 계정으로 접속합니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| (5)  | 아까 root 계정을 만들 때 입력했던 비밀번호를 입력합니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| (6)  | root 계정 사용자에게 모든 DB에 대한 모든 권한을 주면서, 다양한 환경에서 이용할 수 있도록 접근 방식을 %로 설정합니다.<br><br>도커 내부에 있는 mysql의 기준에서 보면 도커를 실행 중인 사람의 로컬조차 외부에 해당합니다. 따라서 연결을 위해서는 반드시 '%' 혹은 '특정아이피'로 설정할 필요가 있습니다.<br><br>권한 설정과 접근 방식은 각자의 상황에 맞게 설정하는 게 좋습니다.<br>만약 다른 팀원들이 다른 DB 사용자계정을 사용한다면, 해당 계정 생성 및 권한 부여, 접근방식 설정도 같이 진행해야 합니다.                                                                                                                                 |
| (7)  | 방금 진행한 권한 설정들이 즉시 DB에 반영되도록 합니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| (8)  | 사용할 DB를 생성합니다.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |

### 2단계 - 도커 내 DB의 외부 접근 허용하기

도커 터미널에서 mysql 컨테이너 내부의 터미널에 접속하고, 아래와 같이 입력합니다.

```java
docker exec -it [컨테이너 이름] /bin/bash --(1)
find /etc -name "*.cnf" --(2)
nano [복사한 경로] --(3)
bind-address = 0.0.0.0 --(4)
docker restart [컨테이너 이름] --(5)
docker inspect -f '{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' [컨테이너 이름] --(6)
```

<br>

| 구분 | 설명                                                                                                                                                                                                       |
| ---- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| (1)  | 컨테이너 내부의 터미널에 접속합니다. 이미 접속되어 있다면 입력하지 않습니다.                                                                                                                               |
| (2)  | ".cnf"라는 확장자가 붙은 파일의 경로를 찾습니다.<br> - "/etc/mysql/my.cnf"<br>- "/etc/mysql/mysql.conf.d/mysqld.cnf"<br> - "/etc/my.cnf"<br> 등의 경로가 나올 것입니다. 찾았다면 해당 경로를 복사해둡니다. |
| (3)  | 이렇게 입력하면 해당 파일의 편집 화면으로 이동합니다.                                                                                                                                                      |
| (4)  | 이 이 내용을 추가하면 외부에서 도커 내부의 mysql 컨테이너에 접근할 수 있도록 허용합니다.<br>입력이 끝났다면 ctrl+o 입력 > 엔터 > ctrl+x로 빠져 나옵니다.                                                   |
| (5)  | 도커 내 mysql 컨테이너를 재시작합니다.                                                                                                                                                                     |
| (6)  | 모든 컨테이너의 IP 주소를 확인합니다. 여기서 mysql 컨테이너 아이피를 확인 후 어딘가에 기록해 두세요. <br>별도의 설정을 하지 않는 이상 172.17.0.2가 기본값입니다.                                           |

### 3단계 - 프로젝트 내부 세팅하기

여기서는 스프링부트를 기준으로 진행합니다.<br>
application.properties 또는 application.yml 파일을 프로젝트 폴더 안에 생성해야 합니다.<br>스프링부트 스타터나 인텔리제이 등으로 프로젝트를 생성했다면 이미 만들어져 있을 겁니다.<br>이미 있다면 다시 만들 필요 없습니다.

<br>

application.properties과 application.yml의 역할 및 기능은 동일하고, 둘 다 있으면 충돌이 발생할 수 있으므로, 둘 중 하나만 있으면 됩니다.<br>그냥 입력 방식의 차이니까 각자 선호하는 쪽을 선택하면 됩니다.

<br>

application.yml로 한다면

```java
>> application.yml
spring:
  datasource:
    driver-class-name: com.mysql.cj.jdbc.Driver
    url: jdbc:mysql://[컨테이너IP]:3306/[DB이름]?serverTimezone=Asia/Seoul&useUnicode=true
    username: root
    password: 1111
```

<br>

application.properties로 한다면

```java
>> application.properties
spring.datasource.driver-class-name=com.mysql.cj.jdbc.Driver
spring.datasource.url=jdbc:mysql://[컨테이너IP]:3306/[DB이름]?serverTimezone=Asia/Seoul&useUnicode=true
spring.datasource.username=root
spring.datasource.password=1111
```

**방금 전에 기록해 둔 mysql 컨테이너 IP를 여기서 쓰시면 됩니다.**

### 5단계 - docker-compose 만들기(선택)

프로젝트 폴더의 디렉터리 루트(가장 밖. 프로젝트 폴더 들어오면 바로 보이는 곳)에 docker-compose.yml 파일을 생성합니다.<br>
당장 이게 없어도 연결은 가능하지만 협업을 위해서는 통일된 세팅을 하는 것이 좋습니다.<br>
이것은 그 통일된 세팅을 하는 과정입니다.

```java
>> docker-compose.yml
version: '3.8'

services:
  db:
    image: mysql:latest
    container_name: [컨테이너 이름]
    environment:
      MYSQL_ROOT_PASSWORD: 1111
      MYSQL_DATABASE: [DB이름]
      MYSQL_USER: user
      MYSQL_PASSWORD: password
    ports:
      - "3305:3306"
    volumes:
      - db_data:/var/lib/mysql
    networks:
      - mynetwork

volumes:
  db_data:

networks:
  mynetwork:
    driver: bridge
```

| 구분                        | 설명                                                                                                                                    |
| --------------------------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| services                    | db라는 서비스(MySQL)를 정의합니다.                                                                                                      |
| image                       | MySQL의 최신 버전 이미지를 사용하도록 지정합니다. 다른 버전으로 고정해도 좋습니다.                                                      |
| container_name              | 아까 만들었던 컨테이너 이름을 입력합니다.                                                                                               |
| environment                 | MySQL의 초기 설정을 환경 변수로 지정합니다.                                                                                             |
| MYSQL_ROOT_PASSWORD         | 아까 입력했던 root 계정의 비밀번호를 입력합니다.                                                                                        |
| MYSQL_DATABASE              | 아까 생성했던 DB 이름을 입력합니다.                                                                                                     |
| MYSQL_USER / MYSQL_PASSWORD | root 계정 이외의 다른 DB 사용자를 이용한다면 그 이름과 비밀번호를 입력합니다.                                                           |
| ports                       | 호스트의 3306 포트를 컨테이너의 3306 포트에 매핑하여 외부에서 접근할 수 있도록 설정합니다. 아까 다른 포트로 설정했다면 그것으로 합니다. |
| volumes                     | MySQL 데이터를 호스트의 볼륨에 저장하여 컨테이너가 삭제되더라도 데이터가 유지되도록 합니다.                                             |
| networks                    | MySQL 컨테이너가 다른 서비스와 네트워크를 통해 연결될 수 있도록 설정합니다.                                                             |

### 6단계 - 도커로 DB 공유하기

> 1. 호스트가 도커 허브에서 레포지토리를 생성합니다.<br>레포지토리는 반드시 public으로 해야 합니다.<br>깃허브랑 달리, private로 설정하면 컨트리뷰터 초대가 불가능합니다.
> 2. 도커 터미널에서 다음의 내용을 입력합니다.
>
> - docker login // 그리고 아이디와 비밀번호 입력
> - docker tag [이미지 이름] [도커허브 사용자명]/[도커허브 레포지토리명]:[태그(선택사항)]
> - docker push [도커허브 사용자명]/[도커허브 레포지토리명]:[방금입력한태그(선택사항)]
>
> 3. 그리고 호스트가 레포지토리에서 다른 팀원들을 컨트리뷰터로 초대합니다.
> 4. 팀원들은 도커 터미널에서 다음의 내용을 입력하여 레포지토리에 올라간 이미지를 내려 받습니다.
>
> - docker pull [도커허브 호스트사용자명]/[도커허브 레포지토리명]:[방금입력한태그(선택사항)]

## 3. 결말

막상 공유를 하고 보니, mysql 프로그램만 공유가 되고, 그 안의 DB와 테이블 등은 공유되지 않았다.
<br><br>
조사해보니, DB와 그 내부 데이터는 별도이기 때문에 단순히 DB의 이미지를 공유하는 것만으로는 데이터를 공유할 수 없다고 한다.
<br>

물론 도커를 통해 DB와 그 데이터를 함께 공유하는 방법도 있다고 한다. 바로 DB가 들어 있는 컨테이너 자체를 이미지화 해서 다시 공유하는 것.
<br><br>
하지만 그 경우에는 이미지와 그 안의 데이터가 고정되어 있기 때문에 수정이 잦은 DB에는 적합하지 않는 방식이기 때문에 추천하지 않는다고 한다.
<br><br>
볼륨을 사용해서 데이터를 관리하는 것도 가능하지만, 이미지와 볼륨을 별도로 관리 및 공유해야 한다고 한다.
<br><br>
그런데 이는 내가 원하는 방향이 아니기에 결국 도커를 통한 DB 공유는 포기했다. DB와 프로젝트 전체를 담은 컨테이너를 만들어서 아예 개발환경을 도커 내부로 세팅할 수도 있겠지만, 그건 지나치게 번거롭다.
<br><br>
도커로 DB를 공유한다는 말을 언뜻 들어서 해본 거였는데, 알고 보니 그건 이미 완성된 다수의 프로젝트 여러 개를 도커에 띄우고, 각 프로젝트들이 옆에 있는 도커의 DB를 공유할 때나 쓰는 거였다. 그리고 그 도커는 로컬이 아니라 웹서버에서 돌린다고 한다....
